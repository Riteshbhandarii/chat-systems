About UmbraChat

Hello there! I'm a second-year Data Engineering student, and UmbraChat is a school project I've developed for my Big Data Engineering course. The goal of this project was to apply the principles and technologies learned in the curriculum to build a real-world application that handles data efficiently and scalably.

You might notice UmbraChat embraces the aesthetics of the night. As a fellow night owl, I find that dark mode isn't just a preference; it's a way of life. Plus, it seemed fitting for a platform where connections spark even after the sun goes down.

UmbraChat is designed to be a secure and reliable messaging platform, focusing on the backend infrastructure required to manage a potentially large volume of messages and user interactions. This project allowed me to delve deep into the challenges and solutions associated with big data in a communication context.

Privacy and Security

Security and user privacy are important considerations for UmbraChat:

  • Secure Password Handling: User passwords are securely handled using Django's built-in password hashing mechanisms. This ensures that sensitive information is never stored in plain text.
  • Minimal User Information: In adherence to principles similar to GDPR, UmbraChat is designed to collect and store only the minimal amount of user information necessary for the core functionality of the messaging platform.
  • User Control over Messages: Users have the option to delete their own messages, providing control over their communication history and enhancing privacy.
  • Personalized User Experience: Upon login, UmbraChat loads and displays the user's own information, such as their username or any relevant profile details, ensuring a personalized and relevant experience.
  • GDPR Compliance and Consent: To comply with the General Data Protection Regulation (GDPR), users are required to explicitly consent to the Privacy Policy during the registration process. This ensures that users are informed about how their data is processed and have given their agreement.
  • Privacy Policy Accessibility: A clear and comprehensive Privacy Policy is readily accessible to all users, outlining the data collected, its usage, and users' rights regarding their personal data.

Building UmbraChat: Technical Details

The architecture of UmbraChat incorporates several key components to handle messaging efficiently:

  • Real-time Communication: UmbraChat utilizes WebSockets for bidirectional, real-time communication between users and the server. This allows for instant message delivery without the need for constant polling.
  • Asynchronous Handling: Django Channels extends Django's capabilities to handle WebSockets and other asynchronous tasks. It provides the framework for managing connections and message routing.
  • WebSocket Server: Daphne serves as the ASGI (Asynchronous Server Gateway Interface) server to handle incoming WebSocket connections and pass them to Django Channels for processing.
  • Message Brokering and Caching: Redis is used as a message broker to facilitate communication between different parts of the application, especially for handling WebSocket messages. It can also be used for caching frequently accessed data to improve performance.
  • Backend API: The backend of UmbraChat is built with Django, providing a robust framework for handling user authentication, message creation, retrieval, and other core functionalities through RESTful APIs.
  • Data Storage: Chat messages and user data are stored in PostgreSQL (psql), a robust and scalable relational database known for its reliability and performance. The schema is optimized for efficient querying of conversations.
